# Membership期限通知 一斉メッセージ送信機能 仕様書

## 概要

StaffUserがMembershipの有効期限（expire_day）に基づいて、対象Userに一斉メッセージを送信できる機能を実装する。送信されたメッセージはUser側のMailbox（受信箱）に届き、既読/未読の管理ができる。また、メッセージ送信時にAndroid/iOS両方にプッシュ通知が届く。

---

## 機能要件

### Staff側機能

#### 1. 任意日数設定機能
- Staffは「有効期限までの日数」を複数登録できる（例：30日、14日、7日、3日）
- 日数は数字の自由記述で指定可能
- 登録した日数設定は一覧表示・編集・削除が可能

#### 2. 固定メッセージ設定機能
- 各日数設定に対して、送信する固定メッセージを設定できる
- メッセージは後から編集可能
- 例：30日前 → 「会員期限が30日後に切れます。更新をご検討ください。」

#### 3. 対象User抽出・メッセージ送信機能
- 選択した日数設定に該当するMembershipを持つUserを自動抽出
- 抽出条件: `expire_day - today = 設定した日数`
- 対象Userに固定メッセージを一斉送信
- 送信時、各対象UserのMailboxにメッセージが追加される
- 同時にFirebase Cloud Messaging (FCM) 経由でプッシュ通知を送信

### User側機能

#### 4. Mailbox（受信箱）機能
- Userは受信したメッセージをMailbox画面で一覧表示できる
- メッセージには既読/未読ステータスがある
- メッセージをタップして開くと既読になる
- 未読メッセージの件数がHomePageのアイコンにバッジ表示される

#### 5. HomePageへのMailboxアイコン追加
- HomePageの右上にMailboxアイコン（封筒アイコン）を配置
- 未読メッセージがある場合、アイコンの右上に未読件数バッジを表示
- アイコンタップでMailbox画面へ遷移

#### 6. プッシュ通知
- UserMessageが作成された時、該当UserのデバイスにFCMプッシュ通知が届く
- Android/iOS両対応
- 通知タップでアプリが開く

## データベース設計

### 新規モデル

#### 1. NotificationDaysSetting（通知日数設定）

| フィールド名 | 型 | 説明 |

void main() async {

#### 3. FCMトークン取得・登録
```dart
class FCMService {
  final FirebaseMessaging _messaging = FirebaseMessaging.instance;

  Future<void> initialize() async {
    // 通知許可リクエスト
    NotificationSettings settings = await _messaging.requestPermission(
      alert: true,
      badge: true,
      sound: true,
    );

    if (settings.authorizationStatus == AuthorizationStatus.authorized) {
      // FCMトークン取得
      String? token = await _messaging.getToken();
      if (token != null) {
        await _registerTokenToServer(token);
      }

      // トークン更新時のリスナー
      _messaging.onTokenRefresh.listen(_registerTokenToServer);
    }
  }

  Future<void> _registerTokenToServer(String token) async {
    // バックエンドにトークン登録
    final deviceType = Platform.isIOS ? 'ios' : 'android';
    await http.post(
      Uri.parse('${baseUri}attendances/fcm-token/register/'),
      headers: {'Authorization': 'JWT $accessToken', 'Content-Type': 'application/json'},
      body: jsonEncode({'token': token, 'device_type': deviceType}),
    );
  }
}
```

#### 3. プッシュ通知送信ロジック
```python
# attendances/utils.py
from firebase_admin import messaging

def send_push_notification(user_ids, title, body):
    """
    指定されたUserにFCMプッシュ通知を送信
    """
    tokens = UserFCMToken.objects.filter(
        user_id__in=user_ids,
        is_active=True
    ).values_list('token', flat=True)

    if not tokens:
        return 0

    # FCMメッセージ作成
    message = messaging.MulticastMessage(
        notification=messaging.Notification(
            title=title,
            body=body,
        ),
        tokens=list(tokens),
    )

    # 送信実行
    response = messaging.send_multicast(message)
    return response.success_count
```

#### 4. UserMessage作成時にプッシュ通知送信
```python
# attendances/views.py
from .utils import send_push_notification

class NotificationSettingsViewSet(viewsets.ModelViewSet):
    # ... 他のメソッド ...

    @action(detail=True, methods=['post'])
    def send(self, request, pk=None):
        setting = self.get_object()
        memberships = get_target_memberships(setting.days_before_expire)

        # 重複User除去
        unique_users = {m.user_id: m.user for m in memberships}

        if not unique_users:
            return Response({'success': False, 'message': '対象者がいません。'})

        with transaction.atomic():
            # 各Userにメッセージ作成
            messages = [
                UserMessage(
                    user=user,
                    title='会員期限のお知らせ',
                    content=setting.message,
                )
                for user in unique_users.values()
            ]
            UserMessage.objects.bulk_create(messages)

            # プッシュ通知送信
            send_push_notification(
                user_ids=list(unique_users.keys()),
                title='会員期限のお知らせ',
                body=setting.message[:100]  # 本文は100文字まで
            )

        return Response({
            'success': True,
            'sent_count': len(unique_users),
            'message': f'{len(unique_users)}名のユーザーにメッセージを送信しました。'
        })
```

class NotificationDaysSetting(models.Model):
    """通知日数設定"""
    days_before_expire = models.IntegerField(
        unique=True,
        verbose_name='有効期限までの日数'
    )
    message = models.TextField(
        verbose_name='通知メッセージ'
    )
    is_active = models.BooleanField(
        default=True,
        verbose_name='有効フラグ'
    )
    created_at = models.DateTimeField(auto_now_add=True)
    updated_at = models.DateTimeField(auto_now=True)

    class Meta:
        ordering = ['days_before_expire']
        verbose_name = '通知日数設定'
        verbose_name_plural = '通知日数設定'

    def __str__(self):
        return f'{self.days_before_expire}日前通知'


class UserMessage(models.Model):
    """ユーザーメッセージ（Mailbox）"""
    user = models.ForeignKey(
        'accounts.User',
        on_delete=models.CASCADE,
        related_name='messages'
    )
    title = models.CharField(
        max_length=255,
        verbose_name='タイトル'
    )
    content = models.TextField(
        verbose_name='メッセージ本文'
    )
    is_read = models.BooleanField(
        default=False,
        verbose_name='既読フラグ'
    )
    read_at = models.DateTimeField(
        null=True,
        blank=True,
        verbose_name='既読日時'
    )
    created_at = models.DateTimeField(auto_now_add=True)

    class Meta:
        ordering = ['-created_at']
        verbose_name = 'ユーザーメッセージ'
        verbose_name_plural = 'ユーザーメッセージ'
        indexes = [
            models.Index(fields=['user', 'is_read']),
            models.Index(fields=['user', '-created_at']),

    def mark_as_read(self):
        """既読にする"""
        if not self.is_read:
            self.is_read = True
            self.read_at = timezone.now()
            self.save(update_fields=['is_read', 'read_at'])


class UserFCMToken(models.Model):

### 対象User抽出ロジック

```python
from datetime import date, timedelta

def get_target_memberships(days_before_expire):
    """
    指定日数後に期限が切れるMembershipを取得
    """
    target_date = date.today() + timedelta(days=days_before_expire)

    return Membership.objects.filter(
        expire_day=target_date,
        is_expired=False,
        is_approved=True
    ).select_related('user', 'attendee', 'course')
```

### Permissionクラス

```python
class NotificationSettingPermission(BasePermission):
    """
    通知設定はStaffのみアクセス可能
    """
    def has_permission(self, request, view):
        return request.user.is_authenticated and request.user.is_staff


class MailboxPermission(BasePermission):
    """
    Mailboxは認証済みユーザーが自分のメッセージのみアクセス可能
    """
    def has_permission(self, request, view):
        return request.user.is_authenticated

    def has_object_permission(self, request, view, obj):
        return obj.user == request.user
```

### 1. Firebase Console設定
1. [Firebase Console](https://console.firebase.google.com/) でプロジェクト「wellbee-fitness」を開く

## 注意事項・考慮事項

### 1. 同一Userの重複送信
- 1人のUserが複数のMembershipを持つ場合でも、1回の送信で1通のみ届く
- User単位でユニーク化してから送信

### 2. Mailboxアイコンのパフォーマンス
- HomePageの`initState`で未読件数を取得
- 必要に応じて`setState`で更新
- Mailboxから戻る際に件数を再取得

### 3. FCMトークン管理
- ログイン時にトークン登録
- ログアウト時にトークン削除（is_active=false）
- トークン更新時は自動で再登録

### 4. プッシュ通知の制限
- FCMは一度に最大500デバイスにマルチキャスト可能
- 大量送信時はバッチ処理が必要

### 5. バッチ処理（将来拡張）
- 自動送信機能（毎日定時に対象者へ自動送信）は将来のCelery導入時に実装可能

### 6. 国際化
- メッセージは日本語で固定（多言語対応は将来課題）
- Mailbox画面のラベルは英語/クルド語対応が望ましい（既存UIに合わせる）
